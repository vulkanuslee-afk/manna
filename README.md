# 오늘의 만나 — 안드로이드 앱 프로젝트

웹 버전(`오늘의만나_v13.html`)을 그대로 담은 안드로이드 앱이야.
**하단 배너 광고 + "광고 제거" 인앱결제**가 붙어 있어.

---

## 1. 열기

1. 안드로이드 스튜디오 설치 (없으면 developer.android.com/studio)
2. 이 폴더 압축 풀고 → 안드로이드 스튜디오에서 **Open** → `manna` 폴더 선택
3. 첫 실행 때 Gradle이 라이브러리를 자동으로 받아. 인터넷 필요, 몇 분 걸림
4. Gradle이나 AGP 버전 업그레이드 하라고 뜨면 **Update** 눌러도 돼

폰을 USB로 연결하고 ▶ 버튼 누르면 바로 설치돼서 확인 가능해.
지금 상태로도 잘 돌아가 — **테스트 광고**가 뜰 거야 (구글이 제공하는 가짜 광고).

---

## 2. 출시 전에 반드시 바꿔야 할 3곳

### ① AdMob 앱 ID — ✅ 적용 완료
`ca-app-pub-9045678554149635~5002549817`

### ② AdMob 배너 광고 단위 ID
`app/src/main/res/values/strings.xml` 안

```xml
<string name="admob_banner_id">ca-app-pub-3940256099942544/6300978111</string>
```
→ AdMob에서 만든 **배너 광고 단위 ID**로 교체

> ✅ 내 기기(34CF8156…)는 테스트 기기로 등록되어 있어서, 이 폰에서는 항상 "Test Ad"만 나온다.
> 다른 사람 폰에서는 실제 광고가 정상 노출된다. 폰을 바꾸면 새 기기 ID로 다시 등록할 것.

### ③ 인앱 상품 ID
`BillingManager.java` 맨 위

```java
public static final String PRODUCT_ID = "remove_ads";
```

Play Console → **수익 창출 → 인앱 상품**에서
- 상품 ID: `remove_ads` (위와 똑같이)
- 유형: **일회성 상품(비소모성)**
- 이름: 광고 제거 / 가격: 예) 2,500원

## 3. 광고 제거 버튼 동작

앱 우측 상단에 작은 **"광고 제거"** 알약 버튼이 뜨게 해뒀어.
HTML은 하나도 안 고치고, 앱이 화면 로딩 후에 버튼만 얹는 방식이야.

- **짧게 누르기** → 결제창
- **길게 누르기** → 구매 복원 (폰 바꾸거나 재설치했을 때)
- 구매 완료되면 광고와 버튼이 같이 사라짐

---

## 4. Play Store 올리기

1. **AAB 만들기**: Build → Generate Signed Bundle / APK → **Android App Bundle**
   - 키스토어 새로 만들고 **비밀번호와 .jks 파일 꼭 백업**. 잃어버리면 업데이트 영영 불가
2. Play Console에서 앱 만들기 → AAB 업로드
3. 필수 입력:
   - 앱 아이콘 512×512
   - 스크린샷 최소 2장 (폰에서 캡처하면 돼)
   - 그래픽 이미지 1024×500
   - 짧은 설명(80자) / 자세한 설명(4000자)
   - **개인정보처리방침 URL** — 광고를 붙이면 필수야. GitHub Pages에 페이지 하나 만들어 두면 돼
4. **데이터 안전** 섹션: 광고 SDK가 기기 ID를 쓰니까 그것도 신고해야 함
5. 검토 며칠~2주

## 8. 말씀 듣기 (음성)

앱에서는 **폰에 설치된 음성엔진(구글 TTS)** 을 사용해.
음성 파일을 따로 넣지 않아서 앱 용량이 늘지 않고, 말씀을 추가해도 자동으로 읽혀.

- 브라우저(크롬)에서 열면 → 기존 웹 음성 그대로
- 앱에서 열면 → 폰 내장 음성엔진
- 재생 중에 버튼 다시 누르면 정지, 다른 화면으로 넘어가도 자동 정지

**한국어 음성이 안 나올 때**
"한국어 음성 데이터가 없어"라는 안내가 뜨면서 설정 화면이 열려.
거기서 구글 TTS → 음성 데이터 설치 → 한국어를 받으면 돼. (한 번만 하면 됨)

목소리 속도는 `TtsManager.java`에서 조절:
```java
tts.setSpeechRate(0.92f);  // 낮을수록 느림
tts.setPitch(1.0f);        // 낮을수록 낮은 목소리
```

## 9. 화면 아래 겹침 (제스처 바)

갤럭시 등 최신 기기의 화면 맨 아래 제스처 막대와 광고가 겹치던 문제는
`MainActivity.applySystemBarInsets()` 에서 처리하고 있어.
시스템 바·노치 영역만큼 자동으로 여백을 준다.

기종별로 여백이 어색하면 그 메서드 안의 `v.setPadding(...)` 값을 조정하면 돼.

## 10. Google Play 정책 마감 (2026-08-31)

두 가지 요구사항이 있고, 이 프로젝트는 **이미 충족하도록 갱신됨**.

| 항목 | 요구 | 현재 |
|---|---|---|
| 대상 API 수준 | Android 16 (API 36) 이상 | `targetSdk 36` ✅ |
| 결제 라이브러리 | Play Billing 8 이상 | `billing:8.3.0` ✅ |

- 마감을 넘기면 **새 버전 업로드가 막힐 뿐**, 이미 배포된 앱은 계속 작동한다.
- 필요 시 11월 1일까지 연장 신청 가능(Play Console 정책 상태 페이지).

### Billing 8 변경점 (적용 완료)

`queryProductDetailsAsync`의 콜백이 `List<ProductDetails>` 대신
`QueryProductDetailsResult`를 돌려주도록 바뀌었다.
`BillingManager.loadProduct()`에서 `queryResult.getProductDetailsList()`로 꺼내 쓴다.

### targetSdk 36 확인 사항

Android 16부터 화면을 시스템 바 아래까지 그리는 동작이 기본이 되었다.
이 앱은 `MainActivity.applySystemBarInsets()`에서 이미 처리하고 있어 추가 작업이 필요 없다.
다만 실제 기기에서 **광고가 제스처 바와 겹치지 않는지** 한 번 더 확인할 것.
