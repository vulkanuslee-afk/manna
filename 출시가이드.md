### 방법

# 2단계. 서명 키스토어 + AAB 빌드

## ⚠️ 먼저 읽을 것

여기서 만드는 `.jks` 파일과 비밀번호를 잃어버리면
**이 앱은 영원히 업데이트할 수 없다.** 새 앱으로 다시 올려야 하고, 기존 사용자·리뷰는 전부 날아간다.

만들자마자 **구글 드라이브에 올리고, 비밀번호는 메모 앱에 따로 저장**할 것.

## 키스토어 만들기

1. 안드로이드 스튜디오 상단 **Build → Generate Signed App Bundle / APK**
2. **Android App Bundle** 선택 → Next
3. Key store path 오른쪽 **Create new...** 클릭
4. 입력:

| 항목 | 입력 예시 |
|---|---|
| Key store path | `C:\Users\내이름\manna-key.jks` (기억하기 쉬운 곳) |
| Password / Confirm | 비밀번호 (메모해둘 것) |
| Alias | `manna` |
| Password / Confirm (아래쪽) | 위와 같게 해도 됨 |
| Validity (years) | `30` |
| First and Last Name | 이현민 |
| Organization | 비워도 됨 |
| City / State / Country Code | 화성 / 경기 / **KR** |

5. OK → 다시 Next
6. Build Variants에서 **release** 선택 → Create (또는 Finish)

## 결과 파일 위치

빌드가 끝나면 오른쪽 아래에 알림이 뜬다. 파일 위치는 보통:

```
manna/app/release/app-release.aab
```

이 `.aab` 파일을 Play Console에 올린다.

## 백업 (지금 바로)

- `manna-key.jks` → 구글 드라이브 업로드
- 비밀번호 + Alias(`manna`) → 메모 앱에 저장

# 3단계. Play Console 앱 만들기

play.google.com/console 접속.

## 3-1. 앱 만들기

**앱 만들기** 버튼 → 입력:

| 항목 | 값 |
|---|---|
| 앱 이름 | 오늘의 만나 |
| 기본 언어 | 한국어 |
| 앱 또는 게임 | **앱** |
| 무료 또는 유료 | **무료** |

> ⚠️ **무료/유료는 나중에 바꿀 수 없다.** 광고 + 인앱결제 구조이므로 반드시 **무료**.

체크박스(개발자 프로그램 정책, 미국 수출법) 동의 → 앱 만들기

## 3-2. 스토어 등록정보

왼쪽 메뉴 **성장 → 기본 스토어 등록정보**

| 항목 | 내용 |
|---|---|
| 앱 이름 | 오늘의 만나 |
| 간단한 설명 (80자) | 아래 참고 |
| 자세한 설명 (4000자) | 아래 참고 |
| 앱 아이콘 | 512×512 PNG |
| 그래픽 이미지 | 1024×500 PNG |
| 휴대전화 스크린샷 | **최소 2장** (16:9 또는 9:16) |

### 간단한 설명 (복붙용)

```
지금 내 상황에 맞는 성경 말씀 한 구절. 담아두고, 듣고, 암송하세요.
```

### 자세한 설명 (복붙용)

```
■ 지금 내 마음에 맞는 말씀 한 구절

일이 힘들 때, 불안할 때, 감사할 때.
그 상황에 맞게 미리 골라둔 성경 말씀을 하나씩 보여드립니다.

■ 이런 기능이 있어요

· 상황별 말씀 — 직장, 가정, 불안, 위로 등 카테고리에서 오늘의 말씀을 만납니다
· 말씀 담기 — 마음에 남는 구절을 담아두고 언제든 다시 봅니다
· 말씀 듣기 — 눈이 피로할 때 소리로 들을 수 있습니다
· 암송 연습 — 담아둔 말씀의 핵심 단어를 가리고 외워봅니다
· 기도제목 — 꾸준히 기도할 제목을 적고 응답을 기록합니다
· 말씀 카드 — 마음에 남은 구절을 이미지로 저장해 나눕니다
· 메모 — 묵상하며 든 생각을 구절 옆에 남깁니다

■ 마음 편히 쓰세요

· 회원가입이 없습니다. 이메일도 전화번호도 묻지 않습니다
· 담은 말씀과 기도제목은 내 폰에만 저장됩니다
· 인터넷 없이도 말씀을 볼 수 있습니다

■ 말씀에 대하여

수록된 본문은 성경전서 개역한글판(1961)이며,
2012년 저작권이 만료된 퍼블릭 도메인 저작물입니다.

무작위로 나온 구절이 특별한 계시나 응답이라는 뜻은 아닙니다.
광야의 만나처럼, 매일 새로운 한 구절을 마음에 새기고
천천히 묵상하는 시간이 되길 바랍니다.
```

### 스크린샷 만드는 법

폰에서 앱 실행 → 캡처(전원+음량하강) → 갤러리에서 PC로 옮기기.
추천 화면 4장: **말씀 화면 / 카테고리 목록 / 암송 연습 / 기도제목**

> 스크린샷에 광고가 나온 상태로 찍혀도 상관없다.

## 3-3. 앱 콘텐츠 (심사 필수 항목)

왼쪽 메뉴 **정책 → 앱 콘텐츠**. 아래 순서대로 채운다.

### ① 개인정보처리방침
1단계에서 만든 URL 입력.

### ② 앱 액세스 권한
→ **모든 기능을 제한 없이 사용할 수 있습니다** 선택
(로그인이 없으므로 심사자용 계정이 필요 없다)

### ③ 광고
→ **예, 앱에 광고가 있습니다**

### ④ 콘텐츠 등급
설문에 답하면 자동으로 등급이 나온다.

| 질문 | 답 |
|---|---|
| 카테고리 | **참고자료 / 뉴스 / 교육** |
| 폭력, 성적 콘텐츠, 욕설, 약물 | 전부 **아니요** |
| 사용자 간 소통 기능 | **아니요** |
| 사용자 위치 공유 | **아니요** |
| 디지털 구매 | **예** (광고 제거 상품) |

→ 결과는 **전체이용가** 가 나온다.

### ⑤ 타겟층 및 콘텐츠
- 대상 연령: **만 18세 이상**만 체크
- "아동의 관심을 끌 수 있나요?" → **아니요**

> 만 13세 미만을 선택하면 '가족 정책'이 적용돼 광고 규정이 훨씬 까다로워진다.
> 성인 대상으로 두는 게 간단하다.

### ⑥ 데이터 보안 (제일 헷갈리는 항목)

**"앱에서 필수 사용자 데이터 유형을 수집하거나 공유하나요?"** → **예**

수집 항목은 **광고 SDK 때문에** 신고해야 한다. 아래대로 체크:

| 데이터 유형 | 수집 | 공유 | 목적 | 필수 여부 |
|---|---|---|---|---|
| **기기 또는 기타 ID** | 예 | 예 | 광고 또는 마케팅 | 필수 |
| **앱 상호작용** | 예 | 예 | 광고 또는 마케팅, 분석 | 필수 |
| **대략적인 위치** | 예 | 예 | 광고 또는 마케팅 | 필수 |
| **구매 내역** | 아니요 | — | — | — |

추가 질문:
- 전송 중 데이터가 암호화되나요? → **예**
- 사용자가 데이터 삭제를 요청할 수 있나요? → **예** (앱 삭제로 전부 삭제됨)

> 담은 말씀·기도제목·메모는 **폰에만 저장**되고 외부로 나가지 않으므로 신고 대상이 아니다.
> 결제 정보는 Google Play가 처리하므로 앱이 수집하는 게 아니다.

### ⑦ 정부 앱 / 금융 기능 / 건강 앱
→ 전부 **아니요**

## 3-4. 인앱 상품 등록

왼쪽 메뉴 **수익 창출 → 인앱 상품 → 상품 만들기**

| 항목 | 값 |
|---|---|
| 제품 ID | `remove_ads` ← **코드와 정확히 같아야 함** |
| 이름 | 광고 제거 |
| 설명 | 앱의 모든 광고가 영구적으로 사라집니다. |
| 가격 | 2,500원 (권장) |

만든 뒤 **활성화** 상태로 바꿀 것.

> 상품 ID가 다르면 결제 버튼을 눌러도 "상품 정보를 불러오는 중"에서 멈춘다.

# 4단계. 비공개 테스트 트랙 업로드 ★ 여기서 14일 시작

왼쪽 메뉴 **테스트 및 출시 → 테스트 → 비공개 테스트**

1. **새 버전 만들기**
2. `app-release.aab` 업로드
3. 출시명: `1.0` / 출시 노트: `첫 테스트 버전입니다.`
4. **테스터** 탭 → 이메일 목록 만들기 → 이름 아무거나(예: 만나 테스터)
5. **테스터 지메일 주소를 한 줄에 하나씩** 입력 (12명 이상, 15명 권장)
6. 저장 → **검토 후 출시 시작**

## 심사 대기 (3~4일)

승인되면 이메일이 온다. 그 후 **테스터 목록 아래에 있는 참여 링크(URL)** 를 복사해서
테스터들에게 보낸다. 링크는 이렇게 생겼다:

```
https://play.google.com/apps/testing/com.hyunmin.manna
```

> 테스터는 이 링크에서 **"테스터 되기"** 를 누른 뒤 Play 스토어에서 설치한다.
> 링크를 안 거치고 바로 검색하면 앱이 안 나온다.

# 5단계. 14일 동안 할 일

## 매일 확인

- 테스터 수가 **12명 아래로 떨어지지 않게** 유지 (이탈하면 승인 거절 → 처음부터 14일 다시)
- 중간에 한 번쯤 단톡방에 "설치 유지되고 있나요?" 확인 메시지

## 중간에 업데이트 한 번

`app/build.gradle`에서 숫자만 올리고 다시 AAB 빌드 → 비공개 테스트에 새 버전 업로드.

```gradle
versionCode 2        // 1 → 2
versionName "1.0.1"  // 표시용
```

> 구글이 "테스트 과정에서 무엇을 개선했나요?"를 물으므로,
> 실제로 고친 게 있으면 그걸 올리는 게 좋다.

## 피드백 모으기

테스터들에게 받은 의견을 메모해둘 것. 프로덕션 신청 때 다음을 서술해야 한다.

- 테스터를 어떻게 모집했는지
- 어떤 피드백을 받았는지
- 그 피드백으로 무엇을 바꿨는지

## 이 기간에 마무리할 것

- [ ] 스크린샷 4장 완성
- [ ] 아이콘 512, 그래픽 이미지 1024×500
- [ ] 자세한 설명 다듬기
- [ ] 데이터 보안 항목 저장
- [ ] 인앱 상품 활성화 확인
- [ ] 결제 프로필 등록 (수익 받을 계좌)

# 6단계. 프로덕션 액세스 신청

14일 충족 시 Play Console 대시보드에 신청 버튼이 뜬다.

질문에 답하는 요령:

| 질문 | 답변 방향 |
|---|---|
| 테스터를 어떻게 모집했나요 | 교회 공동체와 지인에게 직접 부탁했다고 사실대로 |
| 어떤 피드백을 받았나요 | 실제 받은 의견 2~3개 구체적으로 |
| 피드백으로 무엇을 바꿨나요 | 실제 수정한 내용 |
| 프로덕션 준비가 됐다고 보는 이유 | 기능이 안정적이고 크래시가 없었다는 점 |

→ 심사 (보통 7일 이내) → 승인되면 **프로덕션** 메뉴가 열린다
→ 프로덕션에 AAB 올리고 출시 → **다시 심사** → 스토어 공개

# 자주 막히는 지점

| 증상 | 원인·해결 |
|---|---|
| 테스터가 링크를 눌러도 앱이 안 보임 | 심사 통과 전이거나, 테스터가 "테스터 되기"를 안 누름 |
| 결제 버튼이 "상품 정보 불러오는 중"에서 멈춤 | 인앱 상품 ID 불일치, 또는 상품이 비활성 상태 |
| 광고가 안 나옴 | AdMob 앱 승인 대기 중일 수 있음(최대 며칠). 테스트 광고는 정상 노출됨 |
| 업로드 시 "이미 사용된 versionCode" | `versionCode` 숫자를 올리지 않음 |
| 서명 오류 | 다른 키스토어로 빌드함. 반드시 처음 만든 `.jks` 사용 |
